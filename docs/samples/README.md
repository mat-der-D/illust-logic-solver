# サンプルデータ集

イラストロジックソルバーのテスト用サンプルデータです。

## ファイル一覧

### 1. 基本的な問題（一意解）

#### `easy_5x5.json`
- **サイズ**: 5×5
- **難易度**: 簡単
- **特徴**: 論理的推論のみで解ける最小構成の問題
- **解の種類**: 一意解
- **想定解答時間**: 1秒未満
- **用途**: 基本的なソルバーの動作確認、論理的推論のテスト

#### `medium_10x10.json`
- **サイズ**: 10×10
- **難易度**: 中程度
- **特徴**: 論理的推論のみで解ける、やや複雑な問題
- **解の種類**: 一意解
- **想定解答時間**: 1〜3秒
- **用途**: 中規模問題の性能テスト、論理的推論の完全性確認

#### `heart_15x15.json`
- **サイズ**: 15×15
- **難易度**: 中程度
- **特徴**: ハート型の絵が浮かび上がる問題
- **解の種類**: 一意解
- **想定解答時間**: 3〜10秒
- **用途**: 実用的なサイズでの動作確認、UI表示の確認

#### `backtrack_required_10x10.json`
- **サイズ**: 10×10
- **難易度**: 難しい
- **特徴**: 論理的推論だけでは解けず、バックトラックが必要な問題
- **解の種類**: 一意解
- **想定解答時間**: 5〜30秒（実装による）
- **用途**: バックトラックアルゴリズムのテスト

## 使用方法

### テストの推奨順序

1. **基本動作確認**:
   - `easy_5x5.json` → 最も基本的なテスト
   - `edge_1x1.json` → 最小サイズのテスト

2. **論理的推論のテスト**:
   - `medium_10x10.json`
   - `heart_15x15.json`

3. **バックトラックのテスト**:
   - `backtrack_required_10x10.json`

4. **エラーケースのテスト**:
   - `multiple_solutions.json` → 複数解検出
   - `no_solution.json` → 矛盾検出

5. **エッジケースのテスト**:
   - `edge_all_filled_5x5.json`
   - `edge_all_empty_5x5.json`

6. **パフォーマンステスト**:
   - `edge_large_50x50.json`

### プログラムでの読み込み例

```rust
use std::fs;

fn load_sample(filename: &str) -> Result<PuzzleData, Box<dyn std::error::Error>> {
    let path = format!("samples/{}", filename);
    let content = fs::read_to_string(path)?;
    let puzzle: PuzzleData = serde_json::from_str(&content)?;
    Ok(puzzle)
}

// 使用例
let puzzle = load_sample("easy_5x5.json")?;
let result = solve(puzzle.row_hints, puzzle.col_hints);
```

## 解答の検証方法

各サンプルファイルの `metadata.solution` フィールドに期待される解答（オプション）が含まれています。ソルバーの出力と比較して正しく解けているか検証できます。
